  - The plan's technical action specified `TypeOrmModule.forFeature([Video])` only, but this under-specifies what `autoLoadEntities: true` actually needs: TypeORM builds entity metadata (including inverse relation properties) eagerly at `DataSource.initialize()`, and it only auto-loads entities that are registered via `forFeature()` somewhere in the module graph being bootstrapped. `Video` has a `@ManyToOne(() => Channel)` relation, and `Channel` has a `@OneToOne(() => User)` relation back, so bootstrapping `WorkerModule` with only `Video` registered threw `TypeORMError: Entity metadata for Video#channel was not found`, then (after adding `Channel`) `TypeORMError: Entity metadata for Channel#user was not found`.
